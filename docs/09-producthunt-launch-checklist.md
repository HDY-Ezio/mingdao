## 发布目标

### 核心目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 当日 Upvotes | 200+ | Top 5 的门槛大约 150-200 |
| 当日排名 | Top 5 of the Day | 获得更多曝光 |
| 新用户注册 | 500+ | 产品转化 |
| 邮件订阅 | 300+ | 长期资产 |
| 付费用户 | 10-20 | 首日收入 |

### 为什么做 ProductHunt

1. **精准流量**: PH 用户是早期采用者，对新产品好奇
2. **高质量外链**: PH 链接有很高的 SEO 权重
3. **一次性曝光**: 一天内集中获得大量关注
4. **社交证明**: "Featured on ProductHunt" 徽章增加信任
5. **投资人关注**: 很多投资人每天刷 PH

## 发布时间选择

### 最佳发布日

- **最佳**: 周二、周三、周四
- **次佳**: 周一
- **避免**: 周五、周六、周日（流量低）
- **避开**: 重大节日、大型科技会议期间

### 最佳发布时间

- ProductHunt 按 **PST (Pacific Standard Time)** 计算每一天
- 新的一天从 **00:01 AM PST** 开始
- 建议在 **00:01 - 01:00 AM PST** 发布，抢占全天时间
- 换算到北京时间：**16:00 - 17:00**（夏令时）/ **17:00 - 18:00**（冬令时）

### 30 天内避开的竞品发布

- 发布前查看近期是否有同类产品上线
- 尽量避开直接竞品的发布日期

## 发布前准备（T-30 天）

### 产品准备

- [ ] 产品功能完整，无明显 Bug
- [ ] 首页清晰传达产品价值（5 秒内看懂是做什么的）
- [ ] 新用户引导流畅（从访问到使用核心功能 < 3 步）
- [ ] 移动端体验良好
- [ ] 加载速度快（LCP < 2.5s）
- [ ] 准备好应对流量峰值（确保 Vercel/Supabase 能扛住）

### 账号准备

- [ ] 创建 Maker 账号（创始人/团队账号）

- [ ] 给其他产品投票和评论（建立账号活跃度）
- [ ] 加入 ProductHunt 社区的 Discord/Slack

### 研究竞品

- [ ] 查看同类产品在 PH 的表现
- [ ] 分析 Top 产品的共同点（标题、描述、素材）
- [ ] 收集成功和失败的案例
- [ ] 找到我们的差异化卖点

- [ ] 列出 50 个可能帮忙投票的朋友/同行

## Upvote 获取策略

### 合法的方式（推荐）

1. **种子用户群**: 提前组织 30-50 个朋友/用户
2. **社交媒体**: 在 Twitter/LinkedIn 分享，让关注者去投票
3. **社区分享**: 在相关社区（Reddit 等）自然分享
4. **邮件列表**: 通知你的订阅用户
5. **优质评论**: 好的评论自然会带来投票
6. **产品本身**: 产品好会有人主动投票

### 不推荐的方式（有风险）

❌ 买 Upvote（会被 PH 检测到，可能被下架）
❌ 让所有人用同一个 IP 投票
❌ 大量新建账号投票
❌ 在 PH 上用小号给自己评论

> **注意**: ProductHunt 有反作弊机制，违规可能导致产品被下架或清零。
> 宁可排名低一点，也不要冒险违规。

## 常见问题与注意事项

### Q1: 自己发布还是找 Hunter 发布？

**建议**: 如果能找到有影响力的 Hunter（粉丝多的）发布更好。
如果没有，自己发布也可以，关键是产品质量和运营。

### Q2: 发布时间有讲究吗？

是的，ProductHunt 按 PST 午夜作为新的一天的开始。
越早发布，当天的时间越多，积累 upvote 的时间越长。

### Q3: 需要多少 upvote 才能进 Top 5？

不一定，取决于当天的竞争激烈程度。
一般来说：
- Top 10: ~100 upvotes
- Top 5: ~150-200 upvotes
- Top 1: 300+ upvotes

### Q4: 发布后多久能看到效果？

流量高峰在发布当天（尤其是 PST 上午）。
后续 2-3 天会有长尾流量。
长期来说，PH 页面本身会有持续的搜索流量。

### Q5: 需要付费推广吗？

不需要。ProductHunt 的排名主要靠 upvote 和评论质量。
如果产品好，自然会被顶上去。
花钱买的不是真实用户，没有长期价值。
